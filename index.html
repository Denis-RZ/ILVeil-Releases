Layered resistance

More than renamed symbols.

Use the layers that match your application. Unsupported methods fall back safely instead of being transformed blindly.

01
3A→VM→7F

Virtual machine

Selected methods become custom bytecode interpreted through protected dispatch, with per-method mappings and hardened target tables.

  • Branching, switches and recursion
  • Instance fields and call targets
  • Void, value and reference returns
02

Managed IL protection

String encryption, symbol renaming, control-flow transforms, integer masking and decoy structures work as coordinated layers.

03

Runtime resistance

Anti-debug and integrity checks can protect startup and selected execution paths with explicit response policies.

04

Release verification

Post-protection checks, application records, logs and compatibility fallbacks keep the output operable—not merely transformed.
